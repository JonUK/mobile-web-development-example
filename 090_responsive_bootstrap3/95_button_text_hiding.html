You must always provide alternative content to identify the purpose of the button, so that it will make sense to users of assistive technologies. In this case, you could add an aria-label attribute on the button itself or add a some content in the button that is only visible to screen readers using the sr-only class.

Modern versions of assistive technologies will announce CSS generated content, as well as specific Unicode characters. To avoid unintended and confusing output in screen readers (particularly when icons are used purely for decoration), hide them with the aria-hidden="true" attribute.
